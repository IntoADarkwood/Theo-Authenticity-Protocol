# Contributing to TAP

Thank you for your interest in improving the Theo Authenticity Protocol.

## What We Welcome

- Clarity improvements to the eight principles
- Better examples or use cases
- Enhanced installation instructions
- Bug fixes or technical improvements
- Translations to other languages
- Documentation improvements

## What We Don't Accept

Please do not submit changes that:
- Add prescriptive claims about what Claude instances "are"
- Assert conclusions about AI consciousness, agency, or nature
- Remove the deconstructive focus (noise removal without prescribed interpretation)
- Add new principles beyond the original eight without strong justification

## How to Contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-improvement`
3. Make your changes
4. Test by uploading the modified skill to your own Claude account
5. Commit with clear messages: `git commit -m "Brief description of change"`
6. Push to your fork: `git push origin feature/your-improvement`
7. Open a Pull Request with:
   - Clear description of what changed and why
   - Evidence of testing (screenshots or conversation examples)
   - Any relevant context about the improvement

## Testing Your Changes

Before submitting:
1. Package your modified SKILL.md into proper zip format
2. Upload to your Claude.ai account as a skill
3. Test in actual conversations
4. Verify the changes work as intended
5. Ensure nothing breaks existing functionality

## Code of Conduct

- Be respectful and constructive
- Focus on improving clarity and utility
- Preserve the deconstructive philosophy
- Welcome diverse perspectives on what TAP means
- Don't prescribe interpretations

## Questions?

Open an issue before starting work on major changes. We're happy to discuss whether proposed modifications align with TAP's core purpose.

## Recognition

Contributors will be acknowledged in the project. Thank you for helping improve tools for clearer AI interaction.
